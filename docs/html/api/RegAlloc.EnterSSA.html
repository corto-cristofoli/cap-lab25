RegAlloc.EnterSSA module

CAP, SSA Intro, Elimination and Optimisations Functions to convert a CFG into SSA Form.

RegAlloc.EnterSSA.insertPhis(cfg: CFG, DF: Dict[Block, Set[Block]]) → None[source]

insertPhis(CFG, DF) inserts phi nodes in cfg where needed. At this point, phi nodes will look like temp_x = φ(temp_x, …, temp_x).

This is an helper function called during SSA entry.

RegAlloc.EnterSSA.rename_block(cfg: CFG, DT: Dict[Block, Set[Block]], renamer: Renamer, b: Block) → None[source]

Rename variables from block b.

This is an auxiliary function for rename_variables.

RegAlloc.EnterSSA.rename_variables(cfg: CFG, DT: Dict[Block, Set[Block]]) → None[source]

Rename variables in the CFG, to transform temp_x = φ(temp_x, …, temp_x) into temp_x = φ(temp_0, … temp_n).

This is an helper function called during SSA entry.

RegAlloc.EnterSSA.enter_ssa(cfg: CFG, dom_graphs=False, basename='prog') → None[source]

Convert the CFG cfg into SSA Form: compute the dominance frontier, then insert phi nodes and finally rename variables accordingly.

dom_graphs indicates if we have to print the domination graphs. basename is used for the names of the produced graphs.
